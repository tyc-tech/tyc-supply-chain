# 采购支出分析

## 触发条件

采购年度支出复盘、品类战略制定、供应商整合决策、采购节约目标拆解、RFQ 策略前研究时触发。

关键词：支出分析、品类支出、供应商整合、价格基准、集中度、采购战略

## 输入要求

- **buyer** (必填): 采购方企业名称
- **category** (可选): 品类关键词（如"冷链物流"、"精密铸件"）— 限定分析范围
- **region** (可选): 供应商地域（如"长三角"）

## 执行流程

### Step 1: 本方上下游识别
- `get_suppliers_and_customers <buyer>` — 本方已合作供应商清单

### Step 2: 供应商深度画像（并发，对 Step 1 的供应商）
对每个供应商：
- `get_company_registration_info <vendor>` — 工商基础
- `get_company_scale <vendor>` — 规模（判断大 / 小供应商）
- `get_financial_data <vendor>` — 财务能力
- `get_products_info <vendor>` — 主营业务（判断品类归属）
- `get_bidding_info <vendor>` — 中标记录（判断客户结构）
- `get_risk_overview <vendor>` — 风险

### Step 3: 市场基准 / 潜在替代供应商
若指定 `category`：
- `search_companies_by_industry_region <category> --region <region>` — 同品类市场
- 对 TOP 20 并发调 `get_company_scale` / `get_risk_overview`，作为基准池

### Step 4: 同行业同品类标杆
- `search_companies_by_ranking` — 行业榜单
- `search_bids <category>` — 同品类招投标市场行情

### Step 5: 整合信号分析
- 单一品类多供应商 → 标记整合机会
- 规模 / 财务悬殊 → 标记议价空间
- 高风险供应商 → 标记替换必要性

## 输出格式

```markdown

> 分析品类: {category / 全品类} · 地域: {region / 全国}
> 出具: {ISO8601}

## 一、供应商池概览

| 维度 | 数据 |
|------|------|
| 已合作供应商数量 | {n} 家 |
| 品类分布（TOP 5） | ... |
| 地域分布（TOP 5） | ... |
| 规模分布 | 大 {n} / 中 {n} / 小 {n} |

## 二、品类支出画像

| 品类 | 供应商数 | 集中度（TOP 3 占比）| 整合潜力 |
|------|---------|-------------------|---------|
| 品类 A | {n} | {ratio} | 高（建议整合至 2 家）|
| 品类 B | {n} | {ratio} | 中 |

## 三、供应商分层（Kraljic 矩阵思路）

| 分层 | 供应商 | 建议策略 |
|-----|-------|---------|
| 战略 | ... | 长期合作 + 联合研发 |
| 杠杆 | ... | 多源竞标 + 议价 |

| 一般 | ... | 集中采购 + 标准化 |

| # | 品类 | 当前 N 家 | 建议合并至 M 家 | 预估节约 | 理由 |
|---|------|---------|----------------|---------|------|
| 1 | ... | 5 | 2 | 12-18% | 规模差异大 / 价差明显 |

## 五、高风险供应商（建议替换）

| 供应商 | 风险项 | 当前占比 | 备选替代 | 切换紧迫度 |
|--------|-------|---------|---------|-----------|
| ... | 失信 / 被执行 / 经营异常 | ... | 市场池 TOP 3 | 紧急 / 本季度 / 年度 |

## 六、市场基准（若指定 category）

| 基准项 | 本方数据 | 行业 TOP 3 平均 | 差距 |
|--------|---------|---------------|------|
| 供应商规模（参保人数）| ... | ... | ... |
| 财务稳健性（资产规模）| ... | ... | ... |
| 合规健康度（风险数）| ... | ... | ... |

## 七、RFQ / 招标建议
- 品类 A: 建议集中 RFQ，3 家参标，预计节约 {ratio}
- 品类 B: 分层议价（战略 / 一般）
- 高风险 5 家: 立即启动替代供应商开发

## 八、年度目标拆解
- 年度节约目标: {amount}
- 按品类拆分: 品类 A {x} / 品类 B {y} / 品类 C {z}
- 里程碑: Q1 整合 → Q2 签约 → Q3 切换 → Q4 复盘
```

## 错误处理

- `suppliers-and-customers` 无数据 → 输出"供应商池未公开，建议结合 ERP 导入"提示
- 品类无法识别 → 标注"品类识别需结合 ERP 数据"
- 市场基准 `_empty` → 省略第六章

## 示例

输入: `buyer = "某大型制造企业"`, `category = "精密铸件"`, `region = "长三角"`
